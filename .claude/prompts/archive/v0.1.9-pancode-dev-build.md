# v0.1.9: Build pancode.dev with PanCode

## Goal

PanCode builds its own marketing website. This is the integration test
that proves the product works. Use real models, real agents, real dispatch,
real multi-step coordination to build a complete landing page.

## The Task

Build a single-page landing page for pancode.dev that communicates:
- What PanCode is (composable multi-agent runtime)
- Key architecture (8 domains, subprocess isolation, local-first)
- Quick start (npx pancode)
- The positioning (for researchers, labs, power users)

## How to Run It

1. Boot PanCode in the test project: `cd /tmp/pancode-dev-site && npx pancode`
2. Switch to capture mode (Shift+Tab to "Capture")
3. Log the tasks: "Build landing page", "Add hero section", "Add architecture diagram", "Add quick start section", "Add responsive CSS", "Review and polish"
4. Switch to plan mode: Let the orchestrator plan the dispatch sequence
5. Switch to build mode: Let the orchestrator dispatch agents to implement
6. Switch to review mode: Let the orchestrator dispatch reviewers to check quality
7. Iterate until the page is complete

## Success Criteria

- Website renders correctly in a browser
- Built entirely by PanCode agents (no human code edits)
- Multiple agents contributed (dev, reviewer, at minimum)
- Task tracking shows the full task lifecycle
- /cost shows total spend
- /runs shows dispatch history
- No crashes, no orphan processes, no hangs

## What This Tests

- Full mode lifecycle (capture → plan → build → review)
- Shadow agents (pre-dispatch exploration)
- Dispatch chains (if implemented)
- Task tools (task_write, task_check)
- Live dispatch board (active cards during build)
- Multi-agent coordination (dev writes, reviewer checks)
- Session continuity (if session spans multiple interactions)
- Model quality (can local models write decent HTML/CSS?)

## Deliverable

- The pancode.dev website files
- Session transcript showing the full build process
- Performance report: total time, total cost, total runs, model quality notes
- Comparison notes: how did PanCode perform vs doing it manually?
- List of product gaps discovered during the build
