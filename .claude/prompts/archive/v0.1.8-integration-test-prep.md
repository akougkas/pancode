# v0.1.8: Integration Test Preparation

## Goal

Prepare PanCode for its first real-world integration test: building the
pancode.dev website. This session sets up the test project, defines the
agent configurations for web development, compares local model quality,
and verifies every component works end-to-end.

## What to Build

### 1. Test Project Setup

Create a fresh project directory for the website build:
```bash
mkdir -p /tmp/pancode-dev-site
cd /tmp/pancode-dev-site
npm init -y
# Install web dev dependencies (Vite, Tailwind, etc.)
```

Initialize PanCode configuration in the project:
```bash
cp ~/.pancode/agents.yaml .pancode/agents.yaml
# Customize agents for web development
```

### 2. Web Development Agent Definitions

Create specialized agents in `.pancode/agents.yaml`:

- **architect**: Plans page structure, component hierarchy, routing
- **frontend**: Writes HTML/CSS/JS, React/Vue components
- **designer**: Reviews visual design, accessibility, responsive layout
- **qa**: Tests pages, checks links, validates output

### 3. Model Quality Comparison

Test dispatch with different local models to find the best for web dev:
- dynamo LM Studio (qwen3.5-35b-a3b): full capability
- mini llama-server (Qwen35-Distilled): lighter weight
- Local Ollama (qwen3:0.6b): shadow agent only

Run identical tasks on each model, compare output quality:
- "Create an HTML landing page with hero section"
- "Add responsive CSS grid layout"
- "Review the HTML for accessibility issues"

### 4. Browser Agent Investigation

Research how to add browser preview capability:
- Can a PanCode agent open a browser via MCP?
- Can we use Playwright/Puppeteer as a tool?
- What Pi SDK tools support browser interaction?

### 5. End-to-End Verification

Run through every PanCode feature with the test project:
- Modes: capture ideas, plan structure, build pages, ask questions, review
- Tasks: task_write for each page, task_check on completion
- Dispatch: dev agent builds, reviewer agent checks
- Shadow: shadow_explore to understand existing files
- Chains: plan → build → review pipeline
- Board: verify all dispatches visible
- Commands: /cost, /runs, /doctor after heavy session

## Deliverable

- Test project ready at /tmp/pancode-dev-site
- Agent definitions tuned for web development
- Model quality comparison results
- List of any issues blocking the v0.1.9 website build
