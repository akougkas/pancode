# v0.1.2: Command Surface Takeover

## Goal

Own the full slash command surface. PanCode wraps ALL Pi native commands
through its own implementations. Users see only PanCode commands. Pi's
native surface is invisible. ~30 commands, categorized by domain, designed
for power users.

## Locked Decision

Full shadow: register PanCode versions of all 18 Pi commands. Each PanCode
wrapper calls the Pi equivalent internally via ctx methods (ctx.newSession(),
ctx.compact(), ctx.fork(), etc.) then adds PanCode-specific behavior.

## What to Build

### 1. Wrap Pi Native Commands

Register PanCode wrappers for every Pi native command. Each wrapper:
- Calls the Pi SDK equivalent internally
- Adds PanCode-specific behavior where relevant
- Uses PanCode branding in output

| Pi Native | PanCode Wrapper | Extra Behavior |
|-----------|----------------|----------------|
| /new | /new | Clear board + temporal memory + reset tasks |
| /compact | /compact | Prune context registry, log compaction to audit |
| /fork | /fork | Checkpoint before fork |
| /tree | /tree | Pass through |
| /session | /session | Add PanCode domain state summary |
| /export | /export | Include dispatch board state in export |
| /copy | /copy | Pass through |
| /resume | /resume | Restore board state from checkpoint |
| /login | /login | Pass through |
| /logout | /logout | Pass through |
| /reload | /reload | Reload PanCode domains + Pi extensions |
| /hotkeys | /hotkeys | Add PanCode shortcuts (Shift+Tab, etc.) |
| /quit | /exit | Multi-phase shutdown (existing) |

### 2. Hide Pi Native Commands from /help

Use shell-overrides.ts pattern (already exists) to hide all Pi native commands
from the help listing. Only PanCode-registered commands appear.

### 3. Categorized /help

Rewrite /help to show commands grouped by domain:

```
SESSION:  /new /compact /fork /tree /session /resume /checkpoint /context /reset
DISPATCH: /runs /batches /stoprun /cost /dispatch-insights
AGENTS:   /agents /skills
OBSERVE:  /audit /doctor /metrics
SCHEDULE: /budget /cluster
DISPLAY:  /dashboard /status /models /settings /theme /mode /reasoning /help /exit
UTILITY:  /export /copy /login /logout /reload /hotkeys
```

### 4. /settings Redesign

Make /settings the power-user configuration surface. Expose all backend knobs:
- Safety mode (suggest/auto-edit/full-auto)
- Default orchestrator model
- Default worker model
- Reasoning preference
- Theme
- Budget ceiling
- Active domains
- Intelligence enabled/disabled
